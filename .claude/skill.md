# skill

## heavy run

`run` always saves to `<reports_dir>/<TICKER>/<DATE>_<MODEL>_<TS>/` and skips
the in-terminal report. Asset type is auto-detected from the ticker; backend
URL uses the provider's canonical endpoint.

```shell
uv run python -m cli.main run \
  --ticker {ticker} --date {analysis-date} \
    --analysts market,social,news,fundamentals \
    --depth 5 --language English \
    --provider anthropic \
    --deep-model claude-opus-4-8 --quick-model claude-sonnet-4-6 \
    --checkpoint --clear-checkpoints
```

```shell
uv run tradingagents analyze
```

## docs site (MkDocs)

Source markdown lives in `docs/`; built HTML lands in `_site/` (gitignored).
`docs/stylesheets/extra.css` widens content to full viewport. Stray `#`/`##`
headings inside agent body sections of `complete_report.md` are auto-demoted
to `####` on write via `_prune_report_headings` in `cli/main.py`; the
standalone backfill tool is `scripts/prune_report_headings.py` (idempotent).

`docs/index.md` and every `docs/<TICKER>/index.md` are generated by
`scripts/build_reports_site.py` from the run folders — never hand-edit them.
After a new run lands, regenerate indices, then build the site.

```shell
# refresh top-level + per-ticker index pages from docs/<TICKER>/<run>/ folders
uv run python scripts/build_reports_site.py

# one-shot static build into _site/
uv run mkdocs build --clean

# live preview at http://127.0.0.1:8000 (rebuilds on save)
uv run mkdocs serve --dirtyreload

# strict build — fails on broken links / nav warnings
uv run mkdocs build --strict

# backfill: prune headings across every existing report
uv run python scripts/prune_report_headings.py

# backfill: rebuild complete_report.md for runs interrupted before
# consolidation (stage files present, no consolidated report). Run before
# build_reports_site.py so the index never links a missing report.
uv run python scripts/reassemble_complete_reports.py
```
